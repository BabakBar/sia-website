# Blog Backlog

Content backlog for the blog. One brief per post. We walk through each,
polish, and expand into an MDX draft in `src/content/posts/`.

**Voice:** first person, 600–1,200 words, one real snippet or diagram max,
no fluff intro, always end on a concrete takeaway. "Engineer's field notes,"
not tutorials.

**Cadence:** 1/week. Draft 2 ahead so you're never scrambling.
