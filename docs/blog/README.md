# Blog Backlog

This blog is not a generic software/AI/cloud content calendar. It is field notes
from building small operating systems for messy real life: factories, cities,
money, bookmarks, job search, and cloud platforms.

The strongest signal is not that I use AI. It is that I use AI, boring
infrastructure, and simple interfaces to remove real operational friction.

## Positioning

**Working thesis:** practical AI, boring infrastructure, and small systems that
make real work easier.

**One-line identity:** I build small operating systems for messy real life:
factories, cities, money, bookmarks, and cloud platforms.

**Voice:** first person, direct, specific, 700–1,300 words. Field notes, not
thought leadership. Every post should include one concrete decision, tradeoff,
failure, or constraint from a real project.

**Style rules:**

- Lead with the real problem, not with a trend.
- Let projects prove the idea; do not invent frameworks first.
- Prefer constraints, rejected options, and what stayed boring.
- Use one snippet, diagram, screenshot, or table at most.
- End with one concrete takeaway an engineer can apply.
- Avoid LinkedIn-flavored phrases: "game changer," "unlock," "revolutionize,"
  "thought leadership," "AI-powered future."

## Privacy Guardrails

Some source material comes from private repos or employer/client contexts. Posts
must be useful without leaking private details.

Safe to publish:

- architecture patterns and lessons learned
- sanitized snippets and diagrams
- public project names and public repo facts
- anonymized private-project details when the person/client cannot be identified
- general product decisions and tradeoffs

Do not publish:

- Continental subscription, network, tenant, internal architecture, or customer
  details
- proprietary factory/process data, private client screenshots, or non-approved
  images
- Bahar/career-ops personal details without explicit permission and
  anonymization
- credentials, private domains, monitoring URLs, raw logs, costs tied to a
  private client, or repo-private implementation details that are not necessary
- performance claims for industrial AI without context, uncertainty, and limits
