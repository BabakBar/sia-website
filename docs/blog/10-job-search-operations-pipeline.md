# A Job Search Is an Operations Pipeline

- **Status:** sensitive / hold until permission and anonymization
- **Type:** agent workflow / human-in-the-loop operations
- **Primary sources:** career-ops
- **Length:** ~1,000 words
- **Core claim:** a job search can be structured like an operations workflow, but
  the human must remain accountable for every application and message.

## Hook

The useful part was not "AI applies to jobs." The useful part was turning a
messy, emotional process into a reviewed pipeline with clear states.

## Evidence

- career-ops: profile config, portal scanning, JD evaluation, tailored CV/PDF,
  tracker, company research, outreach drafts.
- Ethics rule: no automatic submit; human reviews every application.

## Structure

1. Job search as queue, not chaos.
2. Inputs: profile, target roles, locations, compensation, languages.
3. Processing: scan, score, research, tailor, track.
4. Human checkpoints: review, edit, approve, submit.
5. What should never be automated: truth, consent, representation, final send.

## One concrete artifact

An anonymized pipeline state diagram.

## Takeaway

Automation is most useful when it reduces coordination load without taking away
human judgment.

## Risks

- Requires explicit permission from the person involved.
- Must anonymize all personal data, employers, compensation details, and CV text.
